## 📂 금일 산출물 요약 (Deliverables)

### 1\. 📝 요구사항 명세서 & WBS

  * **기능 정의:** 총 **7개의 핵심 기능** (회원가입, 성향분석, 상품추천, 주가조회, AI예측, 토론방, 마이페이지) 도출.
  * **일정 계획:** 기획(1일) -\> 개발(2일) -\> 테스트(1일)의 4일 단기 완성 로드맵 수립.

### 2\. 🗺️ 데이터베이스 설계 (ERD)

  * **구조:** User(사용자)를 중심으로 Product(상품)와 Stock(주식)이 연결되는 구조 설계.
  * **핵심 포인트:**
      * **`UserProduct`** (사용자가 직접 찜함) vs **`Recommendation`** (AI가 추천해줌) 테이블 분리 전략 수립.
      * N:M 관계를 중계 테이블로 풀어내어 확장성 확보.
  * **산출물:** `ERD_Diagram.png` (첨부 완료)

### 3\. 🖼️ 화면 설계 (Wireframe)

  * **컨셉:** 복잡한 금융 정보를 직관적으로 보여주는 **'카드 뉴스'** 형태의 UI 채택.
  * **주요 화면:**
      * **메인:** 개인화된 추천 메시지와 Top 3 상품 노출.
      * **상세:** 주가 차트와 AI 리포트가 한눈에 들어오는 레이아웃.
  * **산출물:** `Wireframe_Images/` (메인, 상세, 로그인 화면 설계도 첨부)

### 4\. 📡 API 명세서 (API Specification)

  * 프론트엔드와 백엔드 간의 데이터 통신 규약 확정.
  * **주요 엔드포인트:**
      * `GET /api/v1/products/recommend/`: 맞춤 상품 추천 로직
      * `GET /api/v1/stocks/:code/predict/`: AI 예측 데이터 조회

## 📝 기획 회고 (Planning Retrospective)

### 🚀 잘한 점 (Keep)

  * **시각적 도구의 적극적 활용:** 말로만 설명하면 놓치기 쉬운 흐름을 \*\*Mermaid(ERD, Flowchart)\*\*와 **Figma**를 통해 시각화하여 팀원 간의 싱크(Sync)를 빠르게 맞췄습니다.
  * **현실적인 목표 설정:** '모든 주식'을 예측하려던 초기 아이디어를 버리고, 제한된 시간 내에 완성도를 높이기 위해 \*\*'코스피 상위 10개 종목'\*\*으로 범위를 좁히는 결단을 내렸습니다.

### 💦 어려웠던 점 (Problem) & 해결 (Solution)

#### 1\. ERD 관계 설정의 어려움

  * **Problem:** '사용자가 직접 찜한 상품'과 'AI가 추천해준 상품'을 하나의 테이블에서 관리할지, 분리할지 결정하는 데 의견이 갈렸습니다. 또한, User와 Product의 **N:M 관계**를 다루는 것이 복잡하게 느껴졌습니다.

  * **Solution:** **Mermaid**를 사용하여 두 가지 구조를 빠르게 그려보고 비교했습니다. 결론적으로 데이터의 성격(능동적 vs 수동적)이 다르므로 확장성을 위해 \*\*`UserProduct`\*\*와 **`Recommendation`** 테이블로 분리하는 것이 명확하다는 합의를 도출했습니다.

#### 2\. 개발 일정(WBS) 추산의 막막함

  * **Problem:** 아직 코드를 작성해보지 않은 상태에서, 각 기능(AI 모델 연동, 차트 구현 등) 구현에 시간이 얼마나 걸릴지 예측하여 간트 차트를 그리는 것이 매우 어려웠습니다. 특히 **AI 모델 연동** 부분은 경험이 적어 시간을 가늠하기 힘들었습니다.

  * **Solution:** 기능을 '회원가입', '게시판' 같은 큰 단위가 아니라, **'API 명세 작성', 'UI 컴포넌트 제작', '데이터 바인딩'** 처럼 작업 가능한 최소 단위로 쪼개어(WBS Level 3) 시간을 산정함으로써 오차 범위를 줄이려 노력했습니다.
